# <span id="HAPGEN">HAPGEN</span>

### . haps--input file & output file

|descrption||

该文件是包含单个核苷酸多态性（SNP）位点的文件格式，每个位点都有两个等位基因。
0、1的含义是指一个个体在该位点上的等位基因。0表示该位点上有第一个等位基因，1表示该位点有第二个等位基因，每个位点只能有一种等位基因，因此.haps文件只需要用0、1来表示每个个体在该位点上的等位基因。|

### . legend--input file & output file
|column name| descrption|
|:----  |:----  |
|第1列| SNP标识符，用于在不同研究之间对位点进行比较|
|第2列| 位点位置，位点在染色体上的位置，通常以基对（bp）为单位表示|
|第3列| 等位基因频率，该位点的第一个等位基因的频率，通常用小数表示|
|第4列| 等位基因频率，该位点的第一个等位基因的频率，通常用小数表示|
### . map--input file
|column name| descrption|
|:----  |:----  |
|第1列| 染色体位置，这一列表示每个位点在基因组上的物理位置，通常以单位为bp（碱基对）表示。物理位置是基于基因组序列的，因此它不受遗传连锁关系的影响|
|第2列| 遗传距离，遗传距离是基于遗传连锁关系的，它反映了两个位点之间的重组频率。遗传距离越大，表示两个位点之间的重组频率越高，即它们之间的遗传距离越远|
|第3列| 遗传距离相对位置，单位是cM，它是直接从遗传图谱中得到的。遗传图谱是一种基于遗传连锁关系的图谱，用于描述基因组上的遗传距离和基因间的相对位置|


### . gen--output file
|column name| descrption|
|:----  |:----  |
|第1列| SNP位点标识符，标识作用|
|第2列| 位点位置，SNP位点在基因组上的位置|
|第3列| 等位基因，以A、G、C、T进行等位基因1、2的标识|
|第4列|  等位基因，以A、G、C、T进行等位基因1、2的标识|
|第5列| 该SNP位点的等位基因情况在这n-5个样本中的分布情况，例如，第一个样本在该位点上的等位基因情况为10，表示该样本在该位点上有一个等位基因1和一个等位基因2，即杂合|
|第n列|同第五列|


### . sample--output file
|column name| descrption|
|:----  |:----  |
|第1列| 样本的第一个ID，通常是一个数字或字母编号，用于标识每个样本|
|第2列|样本的第二个ID，通常也是一个数字或字母编号，用于标识每个样本|
|第3列| 该样本是否缺失数据的标志，通常用0或1表示，0表示数据完整，1表示数据缺失|
|第4列| 该样本的表型信息，通常用0或1表示，0表示不患病或不表现某种特征，1表示患病或表现某种特征|

### . tags--output file
|column name| descrption|
|:----  |:----  |
|第1列|  SNP 在参考基因组上的位置信息,将模拟数据集中的 SNP 与参考基因组上的 SNP 进行比对，从而更好地理解模拟数据集中的遗传变异情况|







# <span id="PLINK">PLINK</span>
### . QC--input file
|column name| descrption|
|:----  |:----  |
|第1列| CHR,该位点所在的染色体编号|
|第2列| BP,该位点在染色体上的物理位置，通常以基对（bp）为单位|
|第3列| SNP,该位点的单核苷酸多态性（SNP）标识符|
|第4列| A1,该位点的主要等位基因（major allele)|
|第5列| A2,该位点的次要等位基因（minor allele）|
|第6列| N：该位点的样本数，即参与该位点分析的样本数量|
|第7列| SE,该位点的标准误差（standard error）|
|第8列| P,该位点的P值，用于评估该位点与研究特征之间的关联程度|
|第9列| OR,该位点的奥氏比（odds ratio），用于评估该位点与研究特征之间的关联程度。|
|第10列| INFO,该位点的信息度（information score），用于评估该位点的质量和可靠性。通常，INFO值越高，表示该位点的质量越好，可信度越高。|
|第11列| MAF,该位点的最小等位基因频率（minor allele frequency），即次要等位基因在样本中的出现频率。|
### . bed--input file
|descrption||
该文件是一个二进制PLINK格式的文件，包含了每个个体在所有SNP位点上的基因型。该文件不能直接查看，需要使用PLINK软件解析。每个SNP位点的基因型由两个比特位（00，01，10）来表示，对应的等位基因分别为REF等位基因、第一个等位基因和第二个等位基因。每个比特位表示一个个体的基因型（00表示纯合REF基因型，11表示纯合第二等位基因型，01表示杂合基因型）。|
### . map--input file
|column name| descrption|
|:----  |:----  |
|第1列| 染色体位置，这一列表示每个位点在基因组上的物理位置，通常以单位为bp（碱基对）表示。物理位置是基于基因组序列的，因此它不受遗传连锁关系的影响。|
|第2列| 遗传距离，遗传距离是基于遗传连锁关系的，它反映了两个位点之间的重组频率。遗传距离越大，表示两个位点之间的重组频率越高，即它们之间的遗传距离越远。|
|第3列| 遗传距离相对位置，单位是cM，它是直接从遗传图谱中得到的。遗传图谱是一种基于遗传连锁关系的图谱，用于描述基因组上的遗传距离和基因间的相对位置。|

### . bim--input file
|column name| descrption|
|:----  |:----  |

|第2列| SNP标识符，可以是任意字符串|
|第3列| 遗传距离，单位为摩根，一般为0|
|第4列| 物理位置，单位为bp（碱基对），一般为实际的基因组物理位置|
|第5列| 等位基因1的名称，一般为字母或数字|
|第6列| 等位基因2的名称，一般为字母或数字|

|第2列| 个体ID，可以是任意字符串|
|第3列| 父亲ID，如果没有父亲，则为0|
|第4列| 母亲ID，如果没有母亲，则为0|

|第6列| 出生日期，格式为YYYYMMDD，如果未知，则为0|


### . clumped--output file
|column name| descrption|
|:----  |:----  |
|第1列| CHR,该位点所在的染色体编号|
|第2列| F：该位点与最显著信号的关联程度，即该位点的LD值（linkage disequilibrium）|
|第3列| SNP,该位点的单核苷酸多态性（SNP）标识符|
|第4列| BP：该位点在染色体上的物理位置，通常以基对（bp）为单位|
|第5列| P：该位点的P值，用于评估该位点与研究特征之间的关联程度|
|第6列| TOTAL：在该位点的LD窗口内共有多少个位点|
|第7列| NSIG：在该位点的LD窗口内有多少个具有显著关联的位点（即P值低于指定阈值的位点）|
|第8列| S05：在该位点的LD窗口内有多少个具有显著关联的位点，且P值低于0.05。|
|第9列| S01：在该位点的LD窗口内有多少个具有显著关联的位点，且P值低于0.01|
|第10列| S001：在该位点的LD窗口内有多少个具有显著关联的位点，且P值低于0.001|
|第11列| S0001：在该位点的LD窗口内有多少个具有显著关联的位点，且P值低于0.0001|
|第12列| SP2：在该位点的LD窗口内具有最显著关联的位点的P值的平方|


# <span id="IMPUTE2">IMPUTE2</span>
### . vcf--input file
|column name| descrption|
|:----  |:----  |
|第1列| 染色体编号，例如 "1"、"2"、"X" 等|
|第2列| 位点位置，即在染色体上的位置|
|第3列| 参考等位基因，即在参考基因组中所定义的该位点的碱基|
|第4列| 可变等位基因，即在该位点上存在的其他可能的碱基|
|第5列| 质量分数，表示该位点的可靠程度|
|第6列| 过滤标记，表示该位点是否通过了质量控制|
|第7列| 包含了一系列的注释信息，例如变异类型、功能影响、频率等|
|第8列| 格式标记，表示后续的样本数据的格式。|
|第9列| 样本数据列，包含了每个样本在该位点上的基因型信息，例如 "0/1" 表示该样本的两个等位基因分别为参考等位基因和可变等位基因。这些样本数据列的列名即为样本的编号。|
|第n列| 同第9列|

### . height--input file
|column name| descrption|
|:----  |:----  |
|第1列| 家庭 ID，通常用于区分不同家庭或亲缘关系群体。在一些 GWAS 研究中，可能会涉及到家系或亲缘关系的考虑，此时 FID 可以用于标识不同的家庭或亲缘关系群体|
|第2列| 个体 ID，通常用于区分不同个体。在 GWAS 中，每个个体都需要有一个唯一的 IID，以便进行个体之间的区分和比较|
|第3列| 身高数据，通常以数字形式表示每个个体的身高值，例如，以厘米为单位。在 GWAS 中，身高数据是表型数据（Phenotype）之一，与基因型数据（Genotype）一起进行关联分析，以寻找与身高相关的遗传变异|
### . covariate--input file
|column name| descrption|
|:----  |:----  |
|第1列| 家庭 ID，通常用于区分不同家庭或亲缘关系群体。在一些 GWAS 研究中，可能会涉及到家系或亲缘关系的考虑，此时 FID 可以用于标识不同的家庭或亲缘关系群体|
|第2列| 个体 ID，通常用于区分不同个体。在 GWAS 中，每个个体都需要有一个唯一的 IID，以便进行个体之间的区分和比较|
|第3列| 个体的性别,在 GWAS 中，个体的性别可能会对研究结果产生影响，例如，某些性别特定的疾病或表型。因此，在进行 GWAS 分析时，通常需要将个体的性别信息包含在 cov 文件中，以便进行相关的调整或分层分析|


### . assoc--output file
|column name| descrption|
|:----  |:----  |
|第1列| SNP位点的 rsID 或者位置信息|
|第2列| 该位点所在的染色体编号|
|第3列| 该位点在染色体上的物理位置|
|第4列| 该位点的第一等位基因（参考等位基因）|
|第5列| 该位点的第二等位基因（变异等位基因）|
|第6列| 该位点的 P 值，表示该位点与 GWAS 分析中使用的表型数据（如身高、体重、疾病等）之间的关联程度|
|第7列| 该位点的 Odds Ratio，表示变异等位基因与 GWAS 分析中使用的表型数据之间的相对风险|
### . pheno--output file
|column name| descrption|
|:----  |:----  |
|第1列| 家系 ID，表示样本所属的家庭或群体|
|第2列| 个体 ID，表示样本的唯一标识符|
|第3列| GWAS 分析中使用的表型数据，通常是身高、体重、疾病等。在 PRSice 中，该列通常用于计算个体的多基因风险评分（PRS）|


### . best--output file
|column name| descrption|
|:----  |:----  |
|第1列| 家系 ID，表示样本所属的家庭或群体|
|第2列| 个体 ID，表示样本的唯一标识符|
|第3列| 样本是否被包括在回归分析中。如果该列为 1，则表示该个体已被纳入回归分析；如果该列为 0，则表示该个体未被纳入回归分析|
|第4列| 经过计算得到的个体多基因风险评分（Polygenic Risk Score），用于预测个体患病的风险。通常是基于 GWAS 发现的遗传变异，通过加权求和计算得到的|
